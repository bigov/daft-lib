
Пример кода с Использованием пирамиды видимости (frustum)

В папке static код, отображающий статическое пространство, нарисованное
один раз с учетом пирамиды видимости, чтобы иметь возможность оценить
ее форму.

В папке dynamic все видимые грани перерисовываются в каждом кадре. Однажды
нарисованные грани остаются в сцене постоянно, даже когда выходят за пределы
видимости - удаление их из кадра не реализовано. Но тем не менее есть
реальная возможность оценить быстродействие движка на чистом Python в условиях
динамически прорисовываемой сцены.

На размещение в кадре группы размером около 5 тысяч кубов уходит около 
0.08 секунды. Для комфортной работы необходимо быстродействие на уровне примерно
0.01 - 0.02 сек. на кадр.

Следовательно, для обесечения качественной работы графики на чистом
Python+pyglet можно вность/удалять в графическом конвеере OpenGL
не более одной тысячи объектов за один кадр.
